## 二、死亡链暴露的三个真实缺口

### 缺口 1：估值引擎不知道"领先本身值钱"

规则 6.2.1：**设卡必须人站在目标节点上才能设**。跑在对手前面 = 他永远卡不到你
前方的路；被反超 = 前方每个咽喉（武关 S10 / 潼关 S11）都变成他的卡位。
**领先就是一张保险单。**

现在的任务估值 = `任务分 − 绕路帧 × 0.227分/帧`，绕路代价只算鲜度和用时分，
**没算"这次绕路会不会把保险单让给对手"**。replay 36 那条山路任务线账面每步
净收益为正，四个任务做完保险单没了，一次性赔 650 分。

自相矛盾处：尾段蹲刷已有"领先时不蹲、保节奏"闸门（V3.11 跟随者主义）——
**蹲刷懂这个道理，任务估值不懂。**

### 缺口 2："30 帧对峙后硬闯"在对手正在赶来时是自杀

对峙上限当初防的是"对手赖在节点不走"。replay 36 的场景是**对手正在赶来**：
此时硬闯 71 帧长边 = 把自己捆好送到他刚立的卡面前。
判据在决策当帧可精确计算：**边长 > 对手到点时间 + 4 帧设卡读条 ⇒ 硬闯必被冻**，
不用赌。

### 缺口 3：对蹲点补卡者，攻坚/削弱是无底洞，强制通行才是解药（规则可证）

- **攻坚/削弱**：拆掉一张卡，对手站原地 ≤3 好果 + 4 帧再立一张；我们过边要 71 帧，
  他有的是时间补。拆一次亏一次（replay 36：6 人手打水漂），无限循环。
- **强制通行（FORCED_PASS）**：规则 6.3.2——时间税**窗口创建时一次锁定**，
  "之后新增设卡不计入本次强制通行时间税"，通行期间按自身倒计时推进**不会被冻**。
  **强通对补卡免疫**：交一次过路费走 VIP 通道，后面再立多少卡都拦不住。
  武关关键关隘税 ≤45 帧 + 路程 ≈ 120 帧——难看，但对比 195 帧冻死+未交付是天壤之别。

现在突破梯队攻坚第一、削弱第二、强通垫底：**对手不在场时正确；
对手蹲在目标节点时前两层是陷阱，应直接走第三层。**
判断条件"卡主是否站在目标节点"是公开信息。

## 三、对 V3.13 体检文档 7 点的逐条评审

**前提问题**：文档引用的行号和"卡主在场不削"纪律在本机代码
（V3.12-latent-mechanics @ feat/latent-mechanics）里**不存在**——它审的是更新的
V3.13 代码。动手前必须先同步两边代码，否则修复打在旧代码上。

| 文档观点 | 评审 |
|---|---|
| #1 出牌 best-response（P0） | **方向完全正确**（对手可负担牌集全公开，加权随机在扔信息）。但 +15~40/局偏乐观：11 局里窗口战从来不是主要败因（replay 36 窗口 1 胜照样输 654），实际 ≈ +5~15。**文档漏了一个加分项**：强通触发 PASS 窗口，出牌变强直接给"解药"保驾——#1 与死亡链修复配套。保留 P0，排死亡链之后 |
| #2 三拍资源追踪 | 同意，基本被 #1 吸收（每拍重算对手可负担集即可） |
| #3 富余人手预探（P1） | 同意：便宜、纯增量、规则可证。省 10~15 帧的隐性价值是**节奏=保险单** |
| #4 窗口值不值得打 | 合理，并入 #1，不单独立项 |
| #5 急策阈值改估值 | 同意文档自己的保留态度——现有阈值校准过，别为改而改。缓 |
| #6 主动养悬赏 | 同意置信度低。回放补证：我们的卡几乎没活到悬赏线，对手拆卡积极。缓 |
| #7 盯对手交付赛跑 | 文档把价值算在用时分（0.117/帧）上，**算错了地方**——盯对手 ETA 的真正价值是节奏防御（缺口 1/2 需要，`_opp_eta` 已在算）。并入死亡链修复 |

文档"不要碰的资产"清单（时间/价值口径分离、1.25× 走廊并集、捆绑估值、
回头迟滞、蹲刷跟随者、马匹预留）——**全部同意，一个都别动。**

# V3.98.20 隐藏图路线契约

## 三局根因

### 03 split corridors：处理站协议被上层覆盖

我方 r243 到 S05，r244-r595 连续收到 `PROCESS_REQUIRED`，直到交付已经
数学死亡后才在 r597 开始处理，最终 0 分。Planner 原本给出 `PROCESS`，
Hybrid 的宫门影子竞速又把它替换成 `MOVE`；这不是路线估值问题，而是协议
硬前置优先级错误。

修复后，非 S02 固定处理站的 `PROCESS` 高于动态墙、转农、宫门竞速和影子
追赶；若服务端仍返回 `PROCESS_REQUIRED`，下一帧无条件补发 `PROCESS`。
对手占用同站时也继续提交 `PROCESS` 排队，避免 `WAIT`/情报被服务端拒绝。

### 04 short gate edges：完成 S02 后折返重复换乘

我方完成 S02、到 S03 后，为支线收益又走回 S02，重复处理和窗口争夺。
地图的 S14 入边只有 4 帧，无法在对手踏边后完成反应设卡，因此后段回 S11
吃任务本身合理；真正的损失是前段回头。

修复后，已处理 S02 时，若不经 S02 的前进路线到 S14 只多不超过 8 帧，
Planner 不得为非 S02 目标再次穿回 S02。真实 S02 任务和转农模式不受影响。

### 05 optional S02：一个冰鉴不值 88 帧绕行

S02 并非必经点。我方为了 S02 的冰鉴走 S01->S02，处理后又回 S01，累计
比直走快线多约 88 帧。虽然该局靠鲜度险胜，但把 S14 先手白送给强敌。

修复后开局比较完整到 S14 ETA，包含天气、固定处理、领取帧和马的有限持续
时间。S02 有快马/短马时允许最多 48 帧战略溢价，因为马还能打开后续任务和
冰鉴走廊；只有普通资源时最多允许 12 帧。于是 03/04 的 44 帧快马路线保留，
05 的 88 帧纯冰鉴支线跳过。

# V3.98.27 规则反证审计

## 核心目标

统一目标不是“永远 rush”或“永远堵人”，而是：

> 在始终保有可证明的设卡先手时最大化我方分数；若墙无法改变交付结果，
> 立即回落得分；若我方已无法交付而对手仍能交付，优先拒止避免直接输局。

## 决策优先级

1. 协议恢复与固定处理：`PROCESS_REQUIRED`、固定处理站和在途状态不可被
   任务、移动截击或宫门竞速覆盖。
2. S02 专门博弈：保留 3.96.34 的献贡/兵争不主动认输；S01/S02 马只有
   在真实 ETA 更快或会改变后续先手时才争。普通资源/任务错峰不作用于 S02。
3. S02 结果分流：我方仍能交付则继续融合策略；仅我方不能交付则拒止；
   双方都不能交付才转为全资源刷任务分。
4. 必经主墙：从本局有向拓扑证明所有起点到宫门路径都经过某个
   `KEY_PASS`，才启用完整 S10 型 Warden；节点编号不是资格条件。
5. 2612 动态堵：无固定主墙时，遍历对手公开路线上的所有可抢汇合点，
   只在我方到达加 5 帧设卡窗不晚于对手时执行。不是只检查 S09。
6. S14 收官：没有更早动态墙、S14 确实不可绕且存在反应/预埋条件时，
   才粘性接入宫门 Warden；任务只能消费设卡先手之外的余量。
7. 04 型回落：S14 入边不足 5 帧且对手能立即拆防 4 时，普通 6-8 帧
   动态税不得覆盖任务、资源或路线；只有单张墙能确定令对手超过 600 帧，
   2612 截击才重新获得主动作优先级。

## 证明公式

我方动作可行性使用保守上界：

```text
my_finish = 当前/目标到 S14 的逐帧最坏天气 ETA
          + 地图配置的固定处理
          + 地图配置的 S14 验核
          + S14 到 S15 的保证路线
          + 2 帧交付

允许控制动作 iff
my_finish + EXIT_PAD(10) + SET_GUARD_PAD(5) <= remaining
```

对手是否被拒止使用乐观下界，所有不确定性都让给对手：

```text
opp_gate_eta = 晴天 + 零障碍/零中转处理
             + 当前及全图未领取马的零绕路加速
             + 合法最早疾行时序

dynamic_delay = min(原地等卡/小分队最快拆,
                    绕开目标节点,
                    边上改去其他邻点后回攻)

opp_finish = opp_gate_eta + dynamic_delay
           + max(2, configured_verify - scout(3) - break_order(3))
           + 最快 S14->S15 + 2 帧交付

denial iff 原本可交付 and 加入本墙后 opp_finish > remaining
```

已有宫门墙的到达时间按并发关系计算：

```text
wait_wall_arrival = max(入边剩余, 卡风化剩余)
reentry_arrival   = 入边剩余 + 改边回攻额外路程
earliest_arrival  = min(wait_wall_arrival, reentry_arrival)
```

不能再把“入边 + 风化”相加。防 4 宫门卡还必须考虑破关令、一拍攻坚和
最快 32 帧强通；不能把 120 帧自然风化当作保证收益。

## 24/25 分数回归结论

- 3.98.24 在 04 型图关闭全部控制，所以能稳定回到得分底盘。
- 3.98.25 恢复 2612 后，把“至少制造 6 帧延误”也允许覆盖得分；该延误
  不改变交付结果，却可能抢掉任务和资源，因此出现分数下降。
- 3.98.26/27 保留 2612，但在最终门不可控图只放行严格拒止；这不是回退
  动态堵，而是把“有一点税”与“能改变胜负”分开。

## 保证边界

可以证明的是：在任务书规则、服务端公开字段完整、地图通过 start/inquire
下发且对手动作遵守规则的前提下，当前所有控制结论都使用对手最强合法反制，
不会再靠漏算马、天气、处理、强通或改边来声称“锁死”。

不能诚实保证的是：S02 完全对称、双方资源与确定性策略相同的零和窗口存在
平局均衡；任意未公开服务端改规则、缺字段或全新对手随机策略也不可能由有限
离线样例数学穷尽。因此发布口径应是“已消除已知规则反例并通过反证矩阵”，
而不是声称对所有未知实现绝对必胜。

## P1① 平分决胜银行（最重，方向性输分）

**规则依据**：
- 7.3：未交付时任务分 = min(累计任务基础分, **80**)，无里程碑；总分只有
  min(task,80) + min(悬赏,25) − 惩罚。鲜度/好果**不进未交付总分**。
- 9.5：同分且 >0 → 决胜阶梯：比鲜度 → 比好果 → 比惩罚，胜方 3 分败方 1 分。
  决胜赢的 3 分与正常碾压赢的 3 分等值。
- 9.2：双方都 0 分才是 0:0（唯一真双输）。

**历史败局**：replay-A 型 80:80 双未交付局，鲜度 65.65 vs 66.76（差 1.11）
输 1:3——当时我们站在未领取的冰鉴旁边。由于未交付分按 15/30 步进量化且
封顶 80，双未交付同分是常态而非巧合。

**旧代码三宗罪**：
1. `FARM_TASK_CAP = 150` 无任务书锚点——80 之后每一分都是死分，转农在
   死分上烧掉的每一帧本该去买鲜度；
2. 转农模式冰鉴零权重（`FARM_RESOURCE_VALUE` 只有马）、不领不用；
3. 任务窗无条件鲜供——触顶后赢窗 0 分却烧好果（第二级决胜资产）。

**修复**（warden.py）：
- `FARM_TASK_CAP` 150 → 80；
- `_farm_ladder_bank_action`：触顶后 ①手中冰鲜度 ≤90 立即入账 →
  ②脚下冰领进库存 → ③≤120 帧可达冰专程去取 → ④RUSH 期急策转护果令
  （死农世界疾行/破关一文不值）→ ⑤停靠保鲜（静止 0.05 最低损耗）；
- `_defense_card`：触顶后任务窗弃权省好果；**S02 码头窗不受影响**
  （先手锁教义优先，鲜供照打）。

**裁判补账**（arena.py）：`result()` 新增 `platformPoints` 与 `ladder`
字段，按 9.2/9.5 建模平台积分（3:0 / 3:1 / 1:1 / 0:0）。`winner` 保持
总分口径不变，不动存量断言。此前 arena 同分一律判 0——整个离线测试体系
对阶梯败局是瞎的，80:80 输鲜度这类局永远不会亮红灯。

## P1② MOBILE 中局死亡补判

三态（安全→保先手最大化 / 我死对活→拒止 / 双死→纯农）原只在 S02 完成帧
锁存一次。MOBILE 得分模式离开 S02 后中途被冻/被锁死，`_score_actions`
走纯 Planner，Planner 对死局只会 deliver——空跑交付线到终场，白扔
min(task,80) 农分和拒止机会。

**修复**（hybrid.py `_recheck_midgame_death`）：每帧用与对手同款的最乐观
下界（晴天/疾行/零中转/资源瞬发）证明自身已死才转态——只认证明，不认
"未证安全"。对手仍活 → deny-only；双死 → farm-only（决胜银行入口）。

## P1③ 强通入拒止证明

6.3.2 强通 RUSH 期同样可用（6.5 只禁小分队）。旧证明在 RUSH 期回落风化
寿命账（防 2 普通卡可达 ~56 帧）当"确定锁死"——对手一套折返强闯即可击穿，
而我们已按锁死烧了好果底仓。

**官方机制**（2026-07-05 主办方确认）：中边冻结的对手须"退回起点另一
邻站→折返→从起点强闯"。PASS 窗是三拍牌局，攻方 RUSH 期持增益可打强行
（克鲜供），证明层必须假设攻方赢窗。

**修复**（warden.py `_forced_bypass_delay`）：对手最快旁路上界 =
2×最便宜其他邻边（RUSH 速）+ 强通税 + 3 帧窗口开销；起点无其他邻边则
维持风化/小队账（冻结无解）。`_mobile_stay_delay` 与
`_active_gate_wall_kills` 双双取 min。副作用（有意）：S14 墙杀早走
在折返可活时改为继续驻守——在场才能应 PASS 窗，交付由死线离场兜底，
拒止压力优先于时间分。

## P1④ planner 对手 ETA 计骑乘

`planner._opp_eta` 边上余量按基准 1000 估——对手公开骑快马时真实 ETA
短 ~17%。这是全部对手 ETA 误差中唯一"低估对手速度→把我们送上被冻长边"
的乐观方向（陷阱汇聚判定 `risk = opp_eta + 4 < our_eta`，replay36 型
180 帧冻死的入口）。

**修复**：`_opp_edge_frames_boosted` 计在身增益（兼容
remainRound/remainingRound 等键名）+ 库存马（3.3.2 当帧生效，宁可高估
对手）。同时 `GUARD_MIN_OPP_ETA` 8 → 6：旧值多出的 2 帧是给未建模骑乘
留的隐性垫，现改为显式建模（规则硬下界=提交 T→T+5 起拦，+1 帧余量）。

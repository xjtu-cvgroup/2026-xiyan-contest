3. 任务规划器 plan() 与分数模型

3.1 分数模型：任务基础分是全场最大杠杆

送达 + 任务 + 用时 三项合计随任务基础分的变化（按原始用时分 25 估算）。 0→90 每 1 点基础分 ≈ 3 分综合收益；90 后骤降 → 策略档位「保 90 冲 110+」。

3.2 plan() 流程

  1. 规划锚点 = 停靠节点（移动中取当前目标节点）
  2. 直达 ETA = 到宫门(含阻挡惩罚: 敌卡+35 / 障碍+10) + 验核6 + 宫门→终点 + 交付2； slack = 600 − 当前帧 − ETA − 安全余量60
  3. 已验核 → deliver（离宫门要重新验核，不再接任务）
  4. slack ≤ 0 → deliver（抢救模式：放弃一切任务）
  5. 逐个估值可领任务（活跃、无主或归我、保护期非对方、不在黑名单）：
    • 资源前置：requiredResourceTypes 任一满足；T06 硬规则兜底需要 1 匹马——没马直接跳过
    • 站位：T04 清障可在障碍节点或相邻节点；其余须到任务节点
    • 读条：有本队探路标记 −3 帧（最低 2）
    • 过期：当前帧 + 赶路 + 读条 ≤ expireRound
    • 绕路 = 去任务点 + 任务点→宫门 − 直达宫门；硬约束 绕路+读条 ≤ slack
    • 估值 = 边际任务收益（按 3.1 曲线）×（对手更近则 ×0.5；对手正在处理→放弃） − (绕路+读条) × 帧价值 0.227/帧（鲜度 0.11 + 用时 0.117，恒计）
  6. 资源提货目标（冰鉴/马匹）与任务同台竞价，净收益最高者胜出
  7. （V3.12）落后时（totalScore < 对手）追加悬赏目标同台竞价：敌方仍有效设卡上挂着的悬赏， 单次攻坚可破（好果坏果上限内）且绕路净收益 > 0 → bounty； 领先时不参与竞价（6.3.3：只有落后方攻破才计分，打了也白打）
  8. 净收益最高且 > 0 → task / resource / bounty；否则 deliver

3.3 马匹预留（horses_reserved）

锁定 1 匹马不骑，当且仅当：非 RUSH 阶段 且 任务基础分 < 110 且 （当前任务列表有可做的耗马任务 或 地图配置会刷 T06 且当前帧 < 350）。
教训：骑掉唯一的马只省 ~2 帧 ≈ 0.5 分，却让两个 T06（各 30 基础分）全部做不了。

4. 主车队决策树 main_action()

退赛 / 已交付？ → 是：无动作
↓ 否
忙碌状态（处理中/休整/强通/验核/窗口争夺）？ → 是：无动作（等结算）
↓ 否
正在路线边上移动 (routeEdgeId)？
是 —— 边上分支
a. 下一跳有敌方有效卡（冻结中）→ 标记削弱目标交给小分队，无动作
↓
b. （V3.12 删看门狗改道：任务书 8.2 移动中只能 WAIT/续走/用马，中边改道 MOVE 是非法动作，该分支从未生效）
↓
c. 无移动增益且马匹 > 预留量 → USE_RESOURCE(快马/短程马)
↓
d. 否则无动作（系统继续推进）
否 —— 停靠分支（按序判断，命中即返回）
  1. 鲜度 < 91 且有冰鉴 → USE_RESOURCE(ICE_BOX)
    优先于一切等待/交付：+10鲜度≈18分，防跌破 90/80/70… 转坏阈值
  2. 在终点 S15：已验核且好果>0且鲜度>0 → DELIVER；否则 WAIT
  3. 在宫门 S14 未验核且计划=deliver：RUSH 阶段 → 急策未用先比较三选一 （V3.12：坏果 ≥2 或好果留底仓外仍有余 → VERIFY_GATE(破关令)近乎免费省3帧； 否则鲜度<100 才打 RUSH_PROTECT）再 VERIFY_GATE；非 RUSH → WAIT
  4. 固定处理站未完成：对手正在处理本站 → 情报空转帧（V3.12：有 INTEL 就 顺手标本站，见下）；否则 PROCESS
  5. 计划=task 且已到任务站位 → CLAIM_TASK（独占对象，不让行）
  6. 主动设卡机会（§6 条件全满足）→ SET_GUARD(防守值6)
  7. 顺路领取（计划=task/resource/bounty 或 slack>15）：冰鉴上限2，快马/短程马/过所/官凭/ 情报各1（V3.12 补上文书与情报）；错峰命中先 WAIT 一帧 → CLAIM_RESOURCE
  8. 尾段蹲刷候选点上等刷新 → 情报空转帧（同上）
  9. 赶路（目标 = 任务点 / 资源点 / 悬赏目标(V3.12) / 未验核→宫门 / 已验核→终点，寻路含阻挡惩罚）：
    下一跳有障碍无卡：好果>1 → CLEAR 否则 WAIT
    下一跳有敌卡 → 突破梯队（§5，悬赏目标同样在此接管攻坚）
    对手正在下一跳读条设卡 / 中边陷阱风险 → 情报空转帧（等卡成型后节点攻坚，绝不上边挨冻）
    畅通 → MOVE(下一跳)
情报空转帧（V3.12）：上述几处"注定 WAIT"的分支改为 先查手里有没有 INTEL——有且目标（计划任务点 / 未验核时的宫门 / 本站，取距离≤15且未标记者） 存在 → USE_RESOURCE(INTEL, 目标)；没有则老实 WAIT。 效果同小分队探路（处理帧−3，最低2），但零人手成本，机会成本≈0。

5. 突破敌方设卡（四级梯队）

教训：曾在 S09 干等 S10 敌卡风化 175 帧直接未交付（80:525）。等待永远是最后的选项。

① BREAK_GUARD 一击必破（无读条）： 攻坚值 = 好果×2 + 坏果×3（各≤2篓）≥ 防守值。投入搜索最小成本：坏果优先（本身近乎废物）， 好果保留 5 篓交付底仓
↓ 果品不够破
② 小分队削弱（slack>0 且非 RUSH 且人手≥2）： SQUAD_WEAKEN 每次防守 −2（重发间隔 12 帧），主车队原地 WAIT，削到能破转 ①
↓ 人手不足 / 截止吃紧
③ FORCED_PASS 强制通行兜底： 时间税 min(50, 15+防守×5)（关键关隘），远好于等风化；守方胜出则休整后重试。 规则限制：上次强通到达的节点不能重复强通
↓ 全部不可行
④ WAIT（仅剩规则限制时）

6. 主动设卡（V3 进攻）

demo 用这一招连赢我们四局。成本 ~5 帧 + 1~3 好果（≈4~7 分）； 对手的任何应对都更贵：强通税 35~50 帧、攻坚烧 2~4 篓果、削弱耗 4+ 人手 + 30 帧。

全部条件（按序短路）：
① 非 RUSH 且 slack > 80（自己交付无忧）
② 对手存在且未交付未退赛
③ 当前节点是咽喉类型 {KEY_PASS 关键关隘, PASS 关隘, MOUNTAIN_PASS 栈道, GATE 宫门}，非 S15
④ 节点当前无有效卡（谁的都算）；同节点重试间隔 40 帧
⑤ 本队有效卡 < 2（上限 2，第 3 张顶掉最早的）
⑥ 对手到本节点 ETA ∈ [8, 150] 帧（8=读条4帧+生效余量；150=再远卡先风化/生悬赏）
⑦ 本节点在对手去宫门的高效路线上（经此绕路 ≤ 15 帧容差）
⑧ 好果 − 底价(关隘/宫门1) − 额外2 > 底仓5（投不满防守值 6 就不投）
↓ 全满足
SET_GUARD(extraGoodFruit=2) → 防守值 6 · 关键关隘防6首次风化 45 帧后 · 阻断对手 MOVE，逼其进入 §5 的代价选择

7. 小分队决策 squad_action()

人手初始 8。服务端 squadAvailable 字段缺失时用本地账本兜底（探路−1 / 削弱−2）。RUSH 阶段禁止新派。

优先级 1 —— SQUAD_WEAKEN(削弱目标)： 主车队被卡/被冻结时由 §4/§5 标记，人手≥2 即发（主车队正在流血，每帧都亏）
↓ 无削弱需求
优先级 2 —— SQUAD_SCOUT(探路)，目标按序：
· 宫门（仅当前帧 ≥ 355：验核最早 ~390 开放，标记只活 45 帧，早派必过期）
· 计划任务点（人未到才探——就地开读条吃不到减时；人手 > 1 才派）
通过条件：目标无本队有效标记 · 同目标重发间隔 25 帧 · 到目标 ETA ≤ 40 帧（否则标记先过期）
效果：探路标记使该点处理帧数 −3（最低2）——宫门验核 6→3，读条任务同理
↓ 无探路需求，人手≥2
优先级 3（V3.12）—— SQUAD_REINFORCE(续防)： 仅 is_behind() 时给自家风化中（防守值<4）的设卡续 +2，且对手 ETA∈[8,150] 仍会经过； 领先时故意不续——没有主动撤卡手段，唯一能做的是不主动把卡续到悬赏 30/60 帧触发线上送对手追分分
↓ 无续防需求，人手≥2
优先级 4（V3.12）—— SQUAD_CLEAR(远程清障)： 去任务/资源/交付目标路上撞见非 T04 障碍就派（主车队不用绕路/自己 CLEAR）； 严格排除自己在做的 T04 目标——那要靠 CLAIM_TASK 才算数，代劳清障只会让该 T04 失败且不重刷

8. 窗口出牌博弈 pick_card()

确定性出牌遇同水平对手会陷入平局链（休整3帧+抑制18帧，镜像局曾因此双双 0 分）。 采用加权随机混合策略，可负担的牌按强度加权：

牌成本克制权重（可负担时）
强行 QIANG_XING有马类/疾行令增益时免费，否则耗 1 马胜献贡3.0（仅增益期入池）
兵争 BING_ZHENG1 护卫行动点（初始4，无其他用途=免费筹码）胜验牒/强行3.0
献贡 XIAN_GONG1 好果，要求鲜度 ≥80（好果>2 才入池）胜验牒/兵争2.0
验牒 YAN_DIE1 文书（过所/官凭；V3.12 起顺路主动领取，不再指望白捡）胜强行1.5
弃权 ABSTAIN0—资源窗口 1.5（低价值少花钱）/ 其他 0.5
多窗口同帧只能出 1 张：优先「计划中的任务窗口」→「宫门 GATE」→ 其他；未出牌的窗口自动按弃权结算，无惩罚

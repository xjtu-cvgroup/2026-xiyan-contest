# S10 守望者（WardenStrategy）——对练工程说明

2026-07-05，exp/v3.96-s10-warden。

## 这是什么

把 2026-07-04/05 三局实战语料里的走廊封锁套路做成**独立完整客户端策略**，
上真实环境当主线的对练靶子：

- 路人女主队 vs 我方 80:677（虚卡 camp + 掐点离场，我方 150 任务分未交付）
- BasicPy vs 路人女主队 568:30（障碍强通抢关 + 要塞实卡，camper 被反杀）
- 主办方确认：中边被卡可改道至起点其他相邻节点（回放 r266/r371 两次实证）

套路三段：**竞速占关**（最短路直奔关键关隘，第 1 帧起小分队远程清掉沿途
与关隘障碍——既买帧数、又拆掉对手"障碍强通 8 帧免冻"的白菜价把手；顺路
只领马与文书）→ **虚卡封锁**（绝不预立卡；对手踏上任何来边立即落防 6
反应卡，卡亡即补，人手全押增援，通行窗口按拍轮转兵争/鲜供防守）→
**判死离场**（对手数学性到不了终点、或触及自身交付死线余量；离场时对手
仍在逼近则落临别卡）。

关隘按节点类型（KEY_PASS × 在我方去宫门最短路上）自动识别，换图不失效。

## 怎么跑（真实环境对练）

两个终端各起一个客户端连同一局：

```bash
# A 座：主线
python3 main.py <playerId_A> <host> <port>

# B 座：守望者（环境变量或命名参数二选一）
LYCHEE_STRATEGY=warden python3 main.py <playerId_B> <host> <port>
python3 main.py --host H --port P --player-id ID --strategy warden
```

日志横幅会打印 `strategy=warden` 便于确认没跑错版本。

## 基线（离线冒烟，2026-07-05）

主线（feat 集成线 tip，含 3.55 边改道）vs 守望者，种子 0/3/7：

| seed | 主线 | 守望者 | margin |
|---|---|---|---|
| 0 | 80（未交付，任务 150） | 660（r577 交付） | −580 |
| 3 | 60（未交付） | 483（r577 交付） | −423 |
| 7 | 60（未交付） | 480（r577 交付） | −420 |

**主线 3/3 未交付。** 对练目标：主线迭代到能稳定交付并赢下它。
按帧数账，主线的解依次是：竞速先到关隘（含障碍强通）、被冻改道自救、
钓卡→改道→对可见卡强通、死线预算封顶所有教义等待。

## 调参旋钮（类属性，对练时可改）

| 旋钮 | 默认 | 说明 |
|---|---|---|
| `GUARD_EXTRA` | 2 | 反应卡额外好果（2→防 6） |
| `FRUIT_RESERVE` | 5 | 好果底仓，低于不立卡/不烧鲜供 |
| `EXIT_PAD` | 25 | 离场死线余量 |
| `OPP_SPEED_MARGIN` | 0.8 | 判死时对手 ETA 折扣（骑马余量） |
| `THREAT_ETA` | 90 | 临别卡的"对手在逼近"判定 |

想要"业余版预立卡 camper"（练主线的强通分支）：在对练前把
`_reactive_guard` 的 `_opp_inbound` 条件放宽为 `_opp_near(..., 999)` 即可。

## W1 刻意不做的

双守望者互撞的对策（各自交付，平局收场）。

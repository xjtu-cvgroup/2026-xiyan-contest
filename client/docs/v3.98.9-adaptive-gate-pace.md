# V3.98.9 自适应墙权节奏

## 核心目标

> 在始终保有设卡先手的前提下，最大化我方分数。

本版不重构三套既有策略，只修正它们之间的一处优先级：隐藏旁路图在
S02 获得先手后，不再立即永久切到 S14 Warden，而是启动一个逐帧领先
预算。2621 动态堵点、Planner 和 Warden 的内部规则均保持原样。

## 领先预算

```text
gate_budget = opponent_gate_eta
              + opponent_visible_process_remaining
              - my_gate_eta
              - guard_margin
```

- `my_gate_eta`：我方真实增益、天气和沿途固定处理站下的保守时间。
- `opponent_gate_eta`：给对手疾行、晴天并忽略未来处理站的极限下界。
- 只加回对手当前已经开始、公开且不可撤销的读条剩余帧。
- 对手已承诺终局路线时余量为 5 帧，否则为 8 帧。

任务或资源的成本不是裸读条，而是：

```text
opportunity_cost = ETA(当前位置 -> 目标 -> S14，含领取/任务读条)
                   - ETA(当前位置 -> S14)
```

只有 `opportunity_cost <= gate_budget` 才允许执行。预算不足时只替换主车队
动作朝 S14 推进，窗口牌和合法小分队动作不受影响。

## 动态堵点

- 能严格证明把对手推过 600 帧的动态墙，继续高于 S14 节奏预算。
- 真正提交设卡会制造延误，继续放行。
- 普通动态墙的绕路成本和每一帧驻守都要消耗 S14 领先；预算耗尽即转门。
- 到达 S14 或对手进入既有威胁圈后，仍按原规则粘性切入 Warden。

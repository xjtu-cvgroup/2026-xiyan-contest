# 荔枝争运战 L1 Windows 调测包指南

## 1. 总体描述

这个包用于在 Windows 本地跑一局 **你的客户端 vs 官方 demo** 的 L1 对战：服务端负责裁判和输出结果，两个客户端连入服务端，对局结束后生成回放、分数和调试摘要，并可用 WebGL UI 播放回放。

解压后主要进入 `调测\` 使用。包内启动件是 Windows 原生 `exe` / `bat`，不需要安装 Java；回放 UI 需要本机可用的 Python。

## 2. 目录结构

| 路径 | 内容 |
| --- | --- |
| `调测\test.bat` | 一键调测入口：启动服务端、官方 client、官方 demo，结束后自动拉起 UI |
| `调测\server\` | 裁判服务端、服务端启动脚本、对局输出目录 |
| `调测\client\` | 玩家 1001 的官方 L1 demo 客户端，后续可替换成你的客户端 |
| `调测\demo\` | 玩家 2002 的官方 L1 demo 对照客户端 |
| `调测\ui\` | WebGL 回放 UI，用于播放 `replay.txt` |
| `image\` | 本指南引用的 UI 截图 |

## 3. 快速启动

**重要**：本包存放路径中**不要有空格**！！

进入 `调测\` 目录后直接双击或在命令行执行：

```bat
test.bat
```

常用启动方式：

| 目的 | 命令 |
| --- | --- |
| 端到端调测（对战 -> 生成回放 -> 打开 UI） | `test.bat` |
| 只启动服务端 | `server\start-server.bat` |
| 只启动回放 UI | `ui\start-ui.bat` |

可选参数和环境变量：

| 用法 | 说明 |
| --- | --- |
| `test.bat 30001` | 使用 30001 端口启动服务端 |
| `test.bat 30000 20260701` | 指定端口和随机种子 |
| `set ROUND_TIMEOUT_MS=200` 后运行 `test.bat` | 调整每回合等待时间 |
| `set NO_UI=1` 后运行 `test.bat` | 只跑对战，不自动打开回放 UI |

## 4. server（`调测\server\`）

`server\` 是裁判服务端目录，核心文件是：

| 文件 | 是什么 |
| --- | --- |
| `lychee-arena-server.exe` | Windows 原生裁判服务端 |
| `start-server.bat` | 单独启动服务端的脚本 |

跑完一局后，服务端会在本目录输出：

| 文件 | 是什么 |
| --- | --- |
| `replay.txt` | 公开回放，给 UI 播放和复盘使用 |
| `data.csv` | 最终分数 |
| `client_debug.txt` | 客户端调试摘要，包含回合状态、动作结果和建议 |
| `log.txt` | 服务端日志 |

需要改端口、种子、日志级别或模式时，主要看 `test.bat` 和 `server\start-server.bat` 里传给服务端的参数：

| 参数 | 含义 | 默认 | 说明 |
| --- | --- | ---: | --- |
| `-p` | 服务端监听端口 | `30000` | 客户端连接的 TCP 端口 |
| `-a` | 每回合动作等待时间 | `500` | 本调测包脚本默认传 `ROUND_TIMEOUT_MS`，未设置时为 500ms |
| `-r` | 本局输出目录 | `.` | `log.txt`、`replay.txt`、`data.csv` 等输出到 `server\` |
| `-c` | 注册超时时间 | `30000` | 等待客户端注册的最长时间，单位毫秒 |
| `-d` | 准备超时时间 | `30000` | 等待客户端 ready 的最长时间，单位毫秒 |
| `--mode` | 启动模式 | `client-debug` | 调测包默认使用客户端调测模式 |
| `--debug-visibility` | 调试可见性 | `full` | 调测包默认输出完整调试信息 |
| `--seed` | 随机种子 | `20260618` | 可通过 `test.bat` 第二个参数覆盖 |

## 5. demo / client（`调测\client\`、`调测\demo\`）

两边默认都是官方 L1 demo：

| 目录 | 默认身份 | 用途 |
| --- | --- | --- |
| `client\` | 玩家 1001，名称 `client-l1` | 先验证链路，之后替换成你的客户端 |
| `demo\` | 玩家 2002，名称 `demo-l1` | 官方对照客户端 |

客户端启动脚本约定 4 个参数：

```bat
start.bat  <playerId>  <host>  <port>  <playerName>
```

你的客户端只要连接 `host:port`，并用传入的 `playerId` 注册即可。替换时通常改 `client\start.bat`，把其中启动 `l1-demo.exe` 的命令换成你自己的程序。

写自己的客户端主要看根目录这些材料：

- 通信协议：`一骑红尘：荔枝争运战 通信协议.md`
- 消息样例：`inquire消息.json`、`start消息.json`
- 规则与评分：`一骑红尘：荔枝争运战 参赛选手任务书.md`
- 地图：`map_config.json`、`map_config.xlsx`、`map_config.jpg`

## 6. ui（`调测\ui\`）

UI 是 Unity WebGL 回放前端，用来播放服务端生成的 `replay.txt`。运行 `test.bat` 时，如果没有设置 `NO_UI=1`，脚本会把 `server\replay.txt` 同步到 UI 的 `StreamingAssets\replay.txt`，然后调用 `ui\start-ui.bat` 打开浏览器。

默认访问地址：

```text
http://127.0.0.1:9091/litchi_delivery_replay/
```



默认是标清启动，如果电脑性能好的话，可以访问这个链接

```text
http://127.0.0.1:9091/litchi_delivery_replay?quality=4k
```

（电脑性能差可能带不起来显示不出来详细数据，如果红框数据确实请选择标清）：

UI 里主要包括：

- 3D 地图回放：展示站点、道路、队伍位置、节点名称和行动过程。
- 播放控制：支持重播、暂停、倍速、进度条和当前回合显示。
- 小地图：右上角显示全局路线、队伍位置和事件点。
- 任务与事件面板：展示已公开、已完成、已失败任务数量，以及回合事件列表。
- 行动卡与结算提示：用卡片展示设卡布防、清障任务刷新、天气预告、资源使用、得分变化等关键事件。
- **重要**：点击右上角的“调测地图”按钮，可以看到详细坐标

UI 涉及的主要资源：

| --- | --- |
| `Build\*.unityweb` | Unity WebGL 运行文件，浏览器加载 UI 用 |

## 7. 常见输出检查

调测结束后先看 `调测\server\`：

| 检查项 | 说明 |
| --- | --- |
| `data.csv` 是否有两名玩家分数行 | 有分数行才说明对局真的结束 |
| `replay.txt` 是否存在 | UI 播放依赖这个文件 |
| `client_debug.txt` 是否存在 | 用来定位客户端动作失败、资源不足、目标无效等问题 |
| `log.txt` 是否有异常 | 服务端连接、注册、ready、回合推进问题都先看这里 |
